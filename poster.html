for the Smart Assembly Line

1 Problem Statement
Introduction

Modern smart factories require real-time visibility, simulation capability and predictive analysis of manufacturing systems involving multiple robots, conveyors and coordinated automation. Traditional manufacturing set-ups lack dynamic simulation and synchronisation between physical and virtual models, limiting process optimisation and predictive maintenance.

Project Goal

To design and implement a fully functional Process Digital Twin of a multi-robot smart manufacturing system integrated with real-time control logic using OPC UA communication and 3DEXPERIENCE simulation environment.

Specifications
  • Hardware: Omron NX/NJ PLC, SCARA Robot, Viper 650 Robot, Conveyor System, ASRS Units ×2, Lift Plate System, Proximity Sensors ×4, Pneumatic Cylinders ×5, Solenoid Valves ×5, SMPS 24V DC
  • Software: Sysmac Studio, DELMIA 3DEXPERIENCE, Control/Build Software
  • Communication: OPC UA Protocol (bi-directional, secure)
  • Response Latency: < 100 ms (physical ↔ virtual)
  • Power Supply: 220V AC → SMPS → 24V DC
2 Functional & Non-Functional Requirements
Functional Requirements
  • Real-Time Data Acquisition — Collect data from PLC, sensors & actuators
  • PLC–Digital Twin Comms — Bi-directional data exchange via OPC UA
  • Real-Time Synchronisation — Accurate mapping between physical system & virtual model
  • 3D Visualisation — Display assembly line in 3DEXPERIENCE environment
  • Simulation & Monitoring — Monitor and simulate operations in real time
  • Virtual Commissioning — Validate system virtually before physical deployment
Non-Functional Requirements
  • Scalability — Add more stations, robots & sensors easily
  • Interoperability — Seamless OPC UA communication across devices
  • Usability — User-friendly monitoring & control interface
  • Maintainability — Easy updates, debugging & modifications
  • Security — Secure data exchange between PLC & digital twin

4 Function Structure Diagram / Functional Architecture

Bi-directional OPC UA communication between the Physical System Layer (PLC + sensors/actuators) and the Digital Twin Layer (3DEXPERIENCE/DELMIA), enabling real-time data flow and virtual control-loop feedback.

5 Finalized Concept

System Layout:

ASRS-1 → Lift Plate → Conveyor → SCARA Robot Station → Viper Robot Station → ASRS-2. The PLC governs all operations; OPC UA synchronises the physical assembly line with the Digital Twin in real time.

7 Circuit Design

220V AC supply converted to 24V DC via SMPS. The Omron PLC receives inputs from position sensors (PS1–PS4) and controls the conveyor motor, stoppers, lift plate and robot trigger pins. Continuous OPC UA communication enables sensor-based pallet tracking.
